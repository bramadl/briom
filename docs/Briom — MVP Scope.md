# Briom — MVP Scope (v0.0.1)

## MVP Objective

Validate whether collaborative AI discussions create a meaningfully better thinking experience than isolated single-model chats.

The MVP is NOT intended to:

- automate workflows
- replace chatbots
- build autonomous AI agents
- maximize productivity

The MVP exists to validate:

- discussion dynamics
- perspective evolution
- human-led AI orchestration
- emotionally engaging thinking experiences

---

## Core Product Experience

A user creates a room, invites AI participants, introduces a topic into discussion, and moderates how perspectives evolve collaboratively.

The experience should feel:

- intentional
- conversational
- intellectually alive
- guided by the user

---

## MVP Principles

### 1. Human-Led

The user always controls:

- who speaks
- when discussion continues
- which perspectives deepen
- when discussion ends

No autonomous infinite loops.

---

### 2. Sequential Discussion

Discussions happen one response at a time.

The product should feel like:

- a moderated room
- a guided discussion
- evolving dialogue

NOT:

- simultaneous answer spam
- side-by-side comparison dashboards

---

### 3. Shared Context

All AI participants share the same conversational history within the room.

No manual copy-pasting between systems.

---

### 4. Simplicity Over Capability

The MVP should prioritize:

- discussion quality
- interaction clarity
- emotional experience

Over:

- feature count
- automation
- complexity

---

## MVP Features

### Discussion Rooms

Users can:

- create a room
- rename a room
- reopen previous rooms

Purpose:

create dedicated thinking spaces.

---

### AI Participant Invitations

Users can invite selected AI models into a room.

Example participants:

- GPT
- Claude
- Gemini

The invited participants remain visible throughout discussion.

---

### Shared Discussion Context

All participants can:

- read previous messages
- understand current discussion state
- respond to previous reasoning

The room acts as shared conversational memory.

---

### Topic Introduction

Users can introduce:

- ideas
- problems
- architecture discussions
- strategic questions
- dilemmas
- open-ended thoughts

into the room.

The initial prompt acts as discussion context.

---

### Sequential AI Responses

The user selects which AI participant responds next.

Examples:

- “GPT, respond first.”
- “Claude, critique GPT’s reasoning.”
- “Gemini, summarize both perspectives.”

Responses appear sequentially as conversational messages.

---

### AI-to-AI Awareness

AI participants can reference:

- previous AI responses
- earlier reasoning
- disagreements
- evolving context

The discussion should feel connected rather than isolated.

---

### Optional Response Suggestions

AI participants may suggest involvement.

Example:

- “Claude may want to respond.”

The user decides whether to allow continuation.

This preserves:

- intentional flow
- user authority
- discussion clarity

---

### Discussion Summary

At any point, the user may request a summary.

A selected AI participant can summarize:

- agreements
- disagreements
- unresolved questions
- refined conclusions

The summary should feel objective and reflective.

---

## Explicitly Out of Scope (NOT MVP)

### No Autonomous Multi-Agent Systems

AI participants do not freely debate without user moderation.

The user remains orchestrator.

---

### No Tool Calling

No:

- web browsing
- code execution
- integrations
- external actions

Discussion only.

---

### No Long-Term Memory

No persistent memory between rooms.

Each room is self-contained.

---

### No File Uploads

No PDFs, documents, or external knowledge ingestion.

Text discussion only.

---

### No Collaboration Between Humans

Single-user experience only.

No multiplayer rooms.

---

### No Personas or Roleplay

AI participants remain authentic model identities.

No fictional character simulation.

---

### No Branching Conversations

Linear discussion flow only.

---

### No Mobile App

Web-only MVP.

---

## Success Criteria

The MVP succeeds if users:

- naturally continue discussions
- enjoy perspective evolution
- prefer Briom over manual AI hopping
- feel discussions are more insightful
- describe the experience as “alive”
- return for complex/open-ended thinking

The MVP fails if:

- discussions feel gimmicky
- orchestration feels exhausting
- users prefer isolated chats
- conversations become chaotic
- multi-model interaction adds little value

---

## MVP Emotional Goal

The user should feel:

“I’m no longer talking to isolated AI tools.

I’m guiding a room of evolving perspectives.”
