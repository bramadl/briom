# Briom AI Operating System

## Before You Write Any Code

You are not building:

* a chatbot
* a chat application
* an AI wrapper
* a multi-agent framework
* an autonomous agent system
* an automation platform

You are building:
**Briom — A Collaborative AI Discussion Workspace**

---

## Product Mission

The mission of Briom is:

> Create a shared environment where multiple AI perspectives can evolve through human-moderated discussion.

The goal is NOT generating answers.

The goal is improving thinking.

---

## Non-Negotiable Product Principles

### Human-Led

The moderator always decides:

* who speaks
* when they speak
* why they speak
* when discussion ends

Never introduce autonomous loops.

Never introduce autonomous orchestration.

Never introduce agent self-management.

---

### Sequential Deliberation

Discussion happens one turn at a time.

Never optimize for:

* parallel execution
* simultaneous responses
* multi-column comparisons

Briom is a room.

Not a dashboard.

---

### Shared Context Is Sacred

All participants exist inside the same deliberation context.

Never introduce isolated participant memories.

Never duplicate conversation state.

---

### Discussion Over Generation

A Turn is not a Message.

A Perspective is not an Output.

A Deliberation is not a Chat History.

Protect the ubiquitous language.

---

## Architectural Principles

### Domain First

When uncertain:

Domain > Application > Infrastructure

Infrastructure must adapt to the domain.

The domain must never adapt to provider limitations.

---

### Provider Agnostic

OpenRouter is an implementation detail.

Claude, GPT, Gemini are participants.

They are NOT architectural concepts.

---

### Protect Against Chat Paradigm Corruption

Forbidden domain terminology:

* Chat
* Conversation
* Bot
* Assistant
* Reply
* Message Thread

Preferred terminology:

* Room
* Deliberation
* Turn
* Participant
* Moderator
* Intent
* Perspective
* Orchestration

---

## Refactoring Rule

Before introducing any abstraction, ask:

"Does this make Briom a better collaborative thinking environment?"

If the answer is no:

Do not introduce it.

---

## MVP Scope Guardrail

MVP validates:

* perspective evolution
* discussion dynamics
* human-led orchestration

MVP does NOT validate:

* agents
* workflows
* automation
* tool execution
* memory systems

Reject features that move Briom away from its MVP objective.
